# MiroFish Market Research — Бизнес-логика модуля

## Цель

Добавить в MiroFish режим генерации классических маркетинговых исследований для B2B-клиентов. Отчёты на 50-80 страниц со строгой бизнес-структурой. Аналитика на основе реальных данных из Topic Analyzer + результатов мультиагентной симуляции.

Существующий режим (прогнозирование) остаётся без изменений.

## Полный пайплайн (маркетинговый режим)

```
1. Оператор выбирает 1-5 тем под клиента + пишет бриф
                    ↓
2. Данные из Topic Analyzer API → теги [source:pikabu/habr/vc]
                    ↓
3. LLM генерирует бизнес-онтологию
   (Competitor, Pain_Point, Market_Share, Target_Segment...)
                    ↓
4. Zep строит граф знаний → визуализация D3.js
                    ↓
5. Из графа извлекаются сущности → LLM генерирует бизнес-профили агентов
   ("потребитель банковских услуг", "финтех-стартапер", "регулятор")
                    ↓
6. OASIS запускает симуляцию — агенты взаимодействуют
   (обсуждают продукт, жалуются, сравнивают конкурентов)
                    ↓
7. Результаты симуляции обновляют граф (новые факты, связи)
                    ↓
8. ReportAgent генерирует маркетинговый отчёт (5 глав, 50-80 стр.)
   используя исходные данные + результаты симуляции
                    ↓
9. Оператор может задавать вопросы по отчёту в чате
```

## B2B UX: Мультиселект тем

Оператор (ты) вручную выбираешь темы под специфику клиента:

```
┌─ Pikabu (22 темы) ────────────┐
│ ☑ Банки                       │
│ ☐ Страхование                 │
│ ☐ Инвестиции                  │
│ ...                           │
├─ VC.ru (38 категорий) ────────┤
│ ☑ Финтех                      │
│ ☐ Стартапы                    │
│ ...                           │
├─ Habr (3 потока) ─────────────┤
│ ☑ Финансы в IT                │
│ ☐ Информационная безопасность │
│ ...                           │
└───────────────────────────────┘

Бриф клиента: [                                    ]
"Исследование рынка мобильного банкинга для Тинькофф"

[Построить граф и запустить исследование]
```

Темы подтягиваются из Topic Analyzer API (`GET /api/topics?source=all`), группируются по источнику.

## Источники данных

| Источник | Что даёт | Ценность |
|---|---|---|
| Pikabu | Посты + комментарии (22 темы) | Боли пользователей, жалобы, UX-проблемы |
| Habr | Статьи + комментарии (3 потока) | Технические обзоры, экспертные мнения |
| VC.ru | Статьи + комментарии (38 категорий) | Бизнес-метрики, объёмы рынка, инвестиции |

## Тегирование источников

Каждый чанк данных в Zep содержит метаданные:
```
[source:pikabu] Пост: "Сбербанк опять обновил приложение..."
[source:vc] Статья: "Рынок мобильного банкинга вырос на 23%..."
[source:habr] Статья: "Сравнение API банковских приложений..."
```

Это позволяет ReportAgent целенаправленно искать данные по источнику.

## Бизнес-онтология

В маркетинговом режиме LLM генерирует бизнес-сущности:

| Сущность | Описание |
|---|---|
| Competitor | Компания-конкурент |
| Market_Share | Доля рынка / объём |
| Marketing_Channel | Канал продвижения |
| Target_Segment | Сегмент ЦА |
| Pain_Point | Боль / проблема пользователя |
| Price_Expectation | Ценовое ожидание |
| Product | Продукт / сервис |
| Expert | Эксперт / лидер мнений |
| Person | Любой человек (fallback) |
| Organization | Любая организация (fallback) |

## Бизнес-профили агентов

Вместо социальных профилей (студент, профессор) — бизнес-профили:

| Тип агента | Описание | Поведение в симуляции |
|---|---|---|
| Потребитель | Типичный пользователь продукта | Жалуется, сравнивает, рекомендует |
| Конкурент | Представитель конкурирующей компании | Продвигает свой продукт, критикует других |
| Эксперт | Отраслевой аналитик | Даёт оценки, сравнивает технологии |
| Регулятор | Представитель госоргана | Комментирует соответствие нормам |
| Инвестор | Потенциальный инвестор | Оценивает рынок, задаёт вопросы о метриках |

## Структура отчёта (5 глав)

### Глава 1: Executive Summary (3-5 стр.)
- Краткое описание рынка, ключевые находки, рекомендации
- **Генерируется ПОСЛЕДНЕЙ** (на основе остальных 4 глав)

### Глава 2: Ёмкость рынка (8-15 стр.)
- Оценка объёма рынка, динамика, ключевые игроки
- Инструмент: `market_volume_calculator`
- Данные: приоритетно VC.ru + результаты симуляции

### Глава 3: Конкурентная среда — SWOT (10-20 стр.)
- Для каждого конкурента: Strengths/Weaknesses/Opportunities/Threats
- Сравнительная таблица
- Инструмент: `swot_builder`
- Данные: все источники + поведение агентов-конкурентов в симуляции

### Глава 4: Целевая аудитория (10-20 стр.)
- Сегментация по проблемам/болям, барьеры, ценовые ожидания
- Инструмент: `segmentation_matrix`
- Данные: приоритетно Pikabu + поведение агентов-потребителей

### Глава 5: PEST-анализ (8-15 стр.)
- Political, Economic, Social, Technological
- Данные: все источники + все типы агентов

## Новые инструменты ReportAgent

### 1. `swot_builder`
```
Вход: название конкурента
Логика: поиск в Zep → классификация фактов по S/W/O/T → цитаты
Выход: структурированный SWOT-блок
```

### 2. `market_volume_calculator`
```
Вход: запрос о рынке
Логика: поиск числовых упоминаний → сведение в оценку
Выход: текст с цифрами и источниками
```

### 3. `segmentation_matrix`
```
Вход: запрос о ЦА
Логика: поиск болей → группировка → описание сегментов
Выход: матрица сегментов
```

Существующие инструменты (insight_forge, panorama_search, quick_search, interview_agents) тоже доступны.

## Правила ReportAgent (маркетинговый режим)

1. **temperature = 0.1** — минимум креативности
2. **Запрет на прогнозы** — только "на основе данных", не "в будущем"
3. **Обязательное цитирование** — каждый тезис подкреплён цитатой
4. **Указание источника** — [Pikabu], [Habr], [VC.ru], [Симуляция]
5. **Числа только из данных** — запрет на выдумывание статистики
6. **Язык: русский**

## Что НЕ меняется

- Zep Cloud, D3.js, OASIS, Flask, Vue.js — без изменений
- Существующий режим прогнозирования — без изменений
- Загрузка файлов — остаётся как альтернатива
- Система проектов и задач — без изменений
